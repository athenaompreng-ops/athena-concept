4.2 Solusi Arsitektur Tiga Jalur ATHENA
Solusi yang benar bukan 'slow queue vs fast queue'. ATHENA menggunakan tiga arsitektur berbeda untuk tiga tier yang berbeda, masing-masing dengan UX yang dirancang untuk ekspektasi waktu yang realistis.

JALUR A — FREE TIER: Client-Side Processing (In-Browser)

Cara kerja: Model ML dijalankan langsung di browser pengguna menggunakan TensorFlow.js. Foto TIDAK dikirim ke server ATHENA sama sekali — Anti-AI shield ringan bekerja 100% di perangkat user.

Waktu proses: 30-90 detik di browser biasa | Persepsi user: Langsung, tidak ada 'menunggu antrian'

Keuntungan ganda 4A: (1) Tidak ada queue. (2) Foto tidak pernah ke server — privacy paradox hilang untuk free tier. (3) Biaya GPU server: Rp 0.

Trade-off: Model yang bisa dijalankan di browser lebih ringan dari CA-PGD penuh. Efektivitas perturbasi ~70-80% vs server-side. Untuk free tier, ini acceptable.

JALUR B — PAID TIER: Server-Side Priority Queue + 4A Shield Penuh

Cara kerja: Upload ke server ATHENA → masuk priority queue → CA-PGD penuh dengan ensemble model → download. User tetap di halaman dengan real-time progress bar WebSocket. Semua 4 lapisan (Anti-AI, Anti-NSFW, Anti-Deepfake, Anti-Training) aktif penuh.

Target waktu proses: 15-45 detik (GPU Replicate.com). User tidak 'menunggu antrian' — mereka menunggu proses, yang sangat berbeda secara psikologis.

Nilai yang dijual: Kualitas perturbasi CA-PGD penuh (100% efektivitas 4A Shield) + kecepatan + resolusi tinggi tanpa batas + Athena Seal (watermark kepemilikan).

JALUR C — BULK / ENTERPRISE: Async + Notifikasi Email/Push

Cara kerja: Upload batch foto (10-500 foto) → user menutup browser → sistem ATHENA proses di background → notifikasi email/push saat selesai → download link.

Target pengguna: UMKM dengan stok foto produk, pengrajin batik/tenun dengan katalog produk, agensi kreatif, fotografer profesional.

Di sini waiting time 10-60 menit BISA diterima — karena user tidak perlu menunggu di depan layar. Ini satu-satunya konteks di mana antrian panjang masuk akal.

6. SOLUSI TEKNIS — CORE ATHENA & 4A SHIELD


6.1 Cara Kerja: Adversarial Perturbation (4A Shield)
ATHENA menyematkan adversarial perturbation — modifikasi piksel tak kasat mata — ke dalam foto pengguna. Modifikasi ini tidak terlihat mata manusia (perubahan nilai piksel kurang dari atau sama dengan 8/255 dari skala 0-255), namun secara matematis mengacaukan proses gradient descent yang digunakan AI untuk 'belajar' dari foto tersebut. Empat layer 4A Shield bekerja secara sinergis.

Alur Kerja ATHENA — Server-Side / Paid Tier (4A Shield Penuh)

1. Foto diupload ke ATHENA — magic bytes & virus scan — format validation
2. File dienkripsi (AES-256) — disimpan temporary di Cloudflare R2 storage
3. Job masuk priority queue (BullMQ) — dikirim ke GPU worker di Replicate.com
4. Python worker menjalankan Compression-Aware PGD terhadap ensemble (ResNet-50 + VGG-19 + ViT-B/16)
5. Layer Anti-AI, Anti-NSFW, Anti-Deepfake, Anti-Training diaplikasikan sesuai profil foto
6. Perturbasi yang paling konsisten di ketiga model dipilih (universal perturbation)
7. Athena Seal (invisible watermark forensik) disematkan — untuk bukti kepemilikan hukum
8. Foto terproteksi tersedia via presigned URL — ATHENA Shield Hash di-generate
9. File asli dihapus otomatis — status dapat diverifikasi via Shield Hash endpoint publik

6.2 Compression-Aware PGD — Mengapa Ini Kunci untuk Konten Lokal
Perturbasi adversarial standar (FGSM, PGD biasa) mudah dihilangkan oleh kompresi JPEG yang otomatis dilakukan Instagram dan TikTok — platform utama yang digunakan pengrajin dan kreator lokal Indonesia. CA-PGD mengatasi ini dengan memodelkan proses kompresi JPEG sebagai bagian dari attack pipeline, sehingga perturbasi ATHENA tetap aktif bahkan setelah foto diposting ke platform sosial.

Teknik	Tahan Kompresi JPEG?	Transfer ke Model Asing?	Waktu Komputasi	Cocok untuk ATHENA?
FGSM (standar)	Tidak	Sedang	Sangat cepat (1-2 detik)	Tidak (terlalu lemah)
PGD standar	Sebagian	Sedang	10-30 detik (GPU)	Oke untuk demo
CA-PGD (single)	Ya	Sedang	20-60 detik (GPU)	Minimum viable
CA-PGD + Ensemble	Ya	Tinggi	30-120 detik (GPU)	Target arsitektur ATHENA
TF.js (client-side)	Sebagian	Tergantung model	30-90 detik (CPU browser)	Free tier ATHENA

6.3 Ensemble Surrogate — Mengapa Tiga Model Berbeda
Transferability problem: perturbasi yang efektif terhadap model A mungkin tidak efektif terhadap model B yang berbeda arsitektur. Dengan menyerang ensemble tiga arsitektur fundamental yang berbeda (CNN konvensional ResNet-50, deep sequential VGG-19, attention-based transformer ViT-B/16), perturbasi yang konsisten di ketiganya memiliki probabilitas transfer tinggi ke model arbitrary manapun yang digunakan scraper — termasuk model proprietary yang tidak bisa diakses publik.

6.4 Athena Seal — Invisible Watermark sebagai Lapisan Kedua
Selain perturbasi adversarial 4A Shield, ATHENA menyematkan Athena Seal — invisible forensic watermark menggunakan teknik steganografi berbasis DCT (Discrete Cosine Transform). Watermark ini tidak terlihat, tahan terhadap cropping ringan dan kompresi, dan berfungsi sebagai bukti kepemilikan digital yang dapat dipresentasikan secara hukum.
Athena Seal relevan secara hukum berdasarkan UU ITE No. 11 Tahun 2008 jo. UU No. 19 Tahun 2016 dan UU Hak Cipta No. 28 Tahun 2014 — keduanya memberikan perlindungan terhadap ekspresi kreatif digital termasuk karya seni dan fotografi.

	Athena Seal untuk Kearifan Lokal

Untuk foto karya kerajinan lokal (batik tulis, tenun ikat, ukiran, dll.), Athena Seal menyematkan metadata kepemilikan yang terikat dengan identitas kreator — membuktikan bahwa karya tersebut adalah milik pengrajin Indonesia, bukan domain publik yang bebas digunakan model AI asing. Ini adalah implementasi konkret dari perlindungan hak kekayaan intelektual budaya lokal.
